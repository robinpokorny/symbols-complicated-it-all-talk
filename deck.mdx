## What for?

```js
const sym = Symbol()

const o = {
  name: "My key is a string",
  [sym]: "My key is a symbol"
}

o[sym]
// -> "My key is a symbol"
```

<Notes>property key = property name | property symbol</Notes>


# Object inspection

---

## symbol vs enumerable

---

```js
const b = Symbol();

const o = { a: "A", [b]: "B" };
```

<Notes>non-enumerable</Notes>

---

## Access

```js
o.a;
// -> A
o[b];
// -> B
```

---

## hasOwnProperty

```js
o.hasOwnProperty("a");
// -> true
o.hasOwnProperty(b);
// -> true
```

---

## keys

```js
Object.keys(o);
// -> ["a"]

Object.entries(o);
// -> [["a", "A"]]
```

### complication #4

---

## assign

```js
Object.assign({}, o);
// -> {a: "a", Symbol(): "B"}

{ ...o };
// -> {a: "A", Symbol(): "B"}
```

---

## JSON

```js
JSON.stringify(o);
// -> `{"a":"A"}`
```

### complication #5

---

## getOwn

```js
Object.getOwnPropertyNames(o);
// -> ["a"]
Object.getOwnPropertySymbols(o);
// -> [Symbol()]
```

---

## Reflect

```js
Reflect.ownKeys(o);
// -> ["a", Symbol()]
```

# Where to use

---

## 1. Hidden properties

```js
const ROLE = Symbol();

const createAdmin = name => ({
  name,
  [ROLE]: "admin"
});

const isAdmin = (user) =>
  user[ROLE] === "admin";
```


---

```js
const { name } = myUser

const { [ROLE] } = myUser
// Fails

const { [ROLE]: userRole } = myUser

const isAdmin = ({ [ROLE]: userRole }) =>
  userRole === "admin";
```

---

## Symbol vs non-enumerable

- In object's copy
- No name collision
- Symbol can be non-enumerable
- Both are unsafe

---

## 2. Meta-data

- Well-known symbols on Symbol object
- `Symbol.iterator`
- written as `@@iterator`
- Option for libraries

---

## Some well-known symbols

- `@@iterator` and `@@asyncIterator`
- `@@toPrimitive`
- `@@replace`, `@@match`, and `@@search`
- `@@isConcatSpreadable`
